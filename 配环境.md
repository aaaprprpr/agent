# 前端环境配置

这份只讲前端怎么跑。后端和模型服务不在这里展开。

## 1. 安装 Node.js

前端需要 Node.js 和 npm。

推荐安装 Node.js LTS 版本：

```text
https://nodejs.org/
```

安装完成后，重新打开终端，检查：

```bash
node -v
npm -v
```

能输出版本号就说明装好了。

## 2. 进入前端目录

在项目根目录下执行：

```bash
cd frontend
```

## 3. 安装依赖

第一次运行前端前执行一次：

```bash
npm install
```

这会根据 `frontend/package.json` 下载依赖，并生成 `node_modules/`。

## 4. 启动前端

```bash
npm run dev
```

正常会看到类似：

```text
Local: http://localhost:5173/
```

浏览器打开：

```text
http://127.0.0.1:5173/
```

## 5. 前端连接后端

当前前端默认请求：

```text
http://127.0.0.1:8020/api/run
```

所以要完整对话，需要后端也已经启动在 `8020` 端口。

如果后端地址改了，可以在 `frontend` 目录下新建 `.env.local`：

```text
VITE_AGENT_API_BASE=http://127.0.0.1:8020
```

改完后重新执行：

```bash
npm run dev
```

## 6. 常见问题

如果提示 `npm` 不是内部或外部命令：说明 Node.js 没装好，或者安装后没有重新打开终端。

如果 `npm install` 很慢：可以换网络，或者稍后重试。

如果端口 `5173` 被占用：终端会提示另一个可用端口，按提示打开那个地址即可。

如果页面能打开但发消息失败：通常是后端没启动，或者后端地址不是 `http://127.0.0.1:8020`。
