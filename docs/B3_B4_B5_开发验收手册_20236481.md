## 3. 启动服务器常驻模型服务

新开一个服务器终端，执行：

```bash
cd /root/siton-tmp/assignment_B/agent/code
source /opt/conda/etc/profile.d/conda.sh
conda activate agent
python llm_fastapi_server.py --model_config ../configs/model.yaml --host 127.0.0.1 --port 8012
```

注意：

- 终端不要关闭；第一次 `/generate` 请求会加载模型，耗时较长是正常现象。
- 默认监听 `127.0.0.1`，比 `0.0.0.0` 更安全。
- 如果需要 API Key，可设置环境变量 `B4_LLM_API_KEY`，并在 `configs/model.yaml` 的 `fastapi.api_key` 中填同一个值。

## 4. 健康检查


```json

```

这个命令只检查服务，不会加载模型。


合格标准：

- `raw_model_output.json.status` 为 `success`；
- `ai_message.json.role` 为 `assistant`；
- `ai_message.json` 中要么有非空 `content`，要么有 `tool_calls`；
- 如果第一阶段任务需要工具，应能看到合法 `tool_calls`。

## 9. 不打开 VSCode 的本地客户端模式（可选）

如果模型服务在服务器上，而你想在本地终端调用它，需要先建立 SSH 本地端口转发：

```bash
ssh -N -L 8012:127.0.0.1:8012 root@<服务器地址> -p <SSH端口>
```

不要把密码、Token 或私钥写进项目文件。

端口转发建立后，本地 `configs/model.yaml` 保持：

```yaml
runtime:
  llm_source: fastapi
fastapi:
  base_url: http://127.0.0.1:8012
```

此时本地 B4 不加载模型，只向服务器 `/generate` 发请求。但 B1/B3/B5 的文件读写会发生在本地项目目录，因此正式提交验收仍建议在服务器项目目录完整运行。

## 10. 常见问题

1. `local model path does not exist`
   - 检查 `configs/model.yaml` 中模型路径是否为 `../../Qwen3.5-4B`。

2. `transformers does not provide Qwen3.5 multimodal loader classes`
   - 检查服务器 `transformers` 版本；当前你反馈的 `5.12.1` 是满足要求的。

3. `/generate` 首次请求很慢
   - 正常，首次会加载模型。

4. 显存不足
   - 先减少 `generation.max_new_tokens` 和 `context.max_input_tokens`；
   - 确认没有其他进程占用 GPU；
   - 保持 `torch_dtype: bfloat16`。

5. B4 输出 JSON 解析失败
   - 查看 `outputs/.../raw_model_output.json` 的 `raw_text`；
   - 如果模型输出 Markdown 或解释文字，保留该失败案例，可作为“模型格式遵循失败分析”的材料。
